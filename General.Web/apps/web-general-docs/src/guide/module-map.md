# 模块地图

General Admin 当前按平台治理、项目执行、经营分析和基础能力组织。

## 平台治理

- 租户管理：维护租户、租户授权、运维入口。
- 用户管理：维护账号、角色、组织归属、状态和密码。
- 角色权限：维护菜单权限、按钮权限、账号范围和数据范围。
- 菜单管理：维护前端路由、菜单树和权限码关系。
- 组织架构：维护组织树和人员归属。
- 配置参数：维护系统名称、登录页标题、默认分页、日志保留、任务记录保留等在线配置。
- 缓存管理：按授权、字典、菜单、组织、角色、租户等缓存域刷新版本。
- 字典管理：维护项目状态、任务优先级、文件分类等业务字典。
- 开放接口：维护外部系统接入应用、密钥和授权。
- 文件管理：维护平台文件、文件源、预览、下载和元数据。
- 回收中心：恢复或彻底删除软删除数据。
- 日志中心：记录登录、请求、页面访问、接口访问排行、页面访问排行和运维上下文。
- 系统监控：查看当前服务进程、CPU、内存、磁盘、运行环境。
- 定时任务：维护 Cron 任务、触发器、执行记录、集群节点和手动执行权限。

## 项目执行

- 项目列表：项目基础信息、状态、优先级、负责人、发起人。
- 项目详情：里程碑、周期、任务、风险问题、文档、成员、工时。
- 我的相关：按负责人、发起人、成员关系聚合。
- PMO 视图：跨项目治理和异常项目跟踪。

## 经营分析

- 经营总览：合同、成本、毛利、回款、年底预测。
- 项目经营：合同链、采购链、回款链、变更链和预警。
- 报表中心：面向经营分析和管理汇报的统计视图。

## 部署运维

- 本地开发：后端、前端、文档站分别启动。
- 全量发布：`Script/package-all.sh` 生成发布产物。
- NUC 部署：`Script/deploy-to-linux.py lan --init` 发布到局域网服务器。
- 数据迁移：`migrate-db.sh` 执行 schema 和种子迁移。
