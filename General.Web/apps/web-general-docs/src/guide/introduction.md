# 产品介绍

General Admin 是一个面向企业内部管理的全栈平台，覆盖平台治理、项目执行、经营分析和部署运维。

它不是单纯的后台模板，而是把租户、用户、角色、菜单、组织数据范围、审计、项目管理和经营指标放在同一套工程体系里。

## 核心定位

![平台首页](/product/home.png)

- 平台中心负责租户、账号、权限、组织、菜单、审计和基础能力。
- 项目中心负责项目列表、任务、里程碑、风险问题、成员协同和项目详情。
- 经营中心负责合同、预算、采购、回款、利润、预警和项目经营下钻。
- 部署脚本负责本地、局域网 NUC、服务器全量发布和数据库迁移。

## 设计原则

- Host 用户负责平台级治理，不直接混同为普通租户用户。
- 进入租户运维时要保留 Host 操作人的审计信息。
- 页面查看权限和管理操作权限分开，能看不等于能新增、修改、删除。
- 数据范围优先尊重组织和角色授权，PMO 这类治理角色可拥有跨项目视角。
- 种子数据要能修复已有演示库，不能只服务第一次初始化。

## 技术组成

- 后端：.NET ABP 单体应用，支持 SQLite、PostgreSQL、MySQL。
- 前端：Vue 3、Vben Admin、Ant Design Vue、pnpm monorepo。
- 文档：VitePress，当前站点只改造 `apps/web-general-docs`。
- 桌面端：Avalonia 壳 + WebView。

## 适用场景

- 企业内部项目管理平台。
- 多租户 SaaS 管理后台。
- 需要平台治理、审计追踪、组织数据范围的中后台系统。
- 需要在小型服务器或 NUC 上快速部署的内部管理系统。
