# 项目执行

项目执行模块用于管理项目从立项、执行到收尾过程中的核心信息。

## 可见性

![项目执行](/product/project.png)

项目列表不是简单按组织过滤。系统会综合以下关系：

- 组织数据范围。
- 当前用户是项目经理。
- 当前用户是项目发起人。
- 当前用户是项目成员。
- 当前用户是 PMO 或管理员。

PMO 默认是跨项目治理角色，应能看到租户内全部项目。PM 默认至少能看到自己负责、发起或参与的项目。

## 核心对象

- 项目：编号、名称、类型、来源、组织、状态、优先级、预算、合同额、回款。
- 任务：负责人、状态、优先级、计划时间、实际工时。
- 里程碑：目标节点、负责人、计划日期、完成状态。
- 风险问题：风险、问题、依赖、假设。
- 成员：用户、项目角色、加入时间、离开时间、历史只读。
- 文档：项目资料、版本和状态。

## 典型页面

- 项目列表：用于筛选、查看和进入详情。
- 项目详情：集中展示项目的执行全貌。
- 我的相关：聚合当前用户相关项目、任务和提醒。
- PMO 概览：面向跨项目异常治理。
