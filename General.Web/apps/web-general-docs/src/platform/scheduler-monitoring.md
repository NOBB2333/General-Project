# 定时任务与监控

定时任务、日志中心和系统监控是平台运维能力的核心。

## 定时任务

定时任务模块不是简单的前端列表，它包含任务定义、触发器、执行记录、集群节点和后台执行器。

当前设计要点：

- 每个任务有 `JobKey`、标题、Cron 表达式、处理器 `HandlerKey`、启停状态和下次执行时间。
- 每个任务至少有一个默认触发器，也可以维护多个触发器。
- 执行记录保存开始时间、结束时间、耗时、状态、触发方式和结果。
- 集群节点通过心跳判断在线状态，节点超过阈值未上报会显示离线。
- 任务执行使用锁租约，避免多节点重复执行同一个任务。
- 手动执行和自动触发区分记录，方便审计。

权限拆分：

- `Platform.Scheduler.View`：查看任务、看板、触发器、执行记录和集群节点。
- `Platform.Scheduler.Manage`：新增、编辑、启停、删除任务和触发器，清理执行记录。
- `Platform.Scheduler.Execute`：手动执行任务。

种子任务示例包括日志清理、项目周报提醒、租户健康巡检。日志清理会读取配置参数中的保留天数和记录保留数。

## 日志中心

日志中心汇总请求审计、页面访问、操作日志、异常日志和 ABP 审计日志。

当前统计能力包括：

- 按分类查看访问、审计、异常和操作记录。
- 按接口 URL 聚合访问次数，得到访问最多的接口。
- 按菜单或页面标题聚合访问次数，得到访问最多的页面。
- 按用户名聚合访问次数，得到最活跃用户。
- 支持按关键字、时间范围、分类筛选。

前端页面访问会以 `NAV` 方式写入请求审计队列；后端接口访问由中间件记录。Host 运维租户时，请求审计会带上运维会话、目标租户和代入用户信息。

## 系统监控

系统监控是实时采样，不是长期时序监控。它读取当前后端进程和宿主机信息，用于快速判断服务状态。

当前指标来源：

- 进程信息：启动时间、运行时长、线程数、句柄数、工作集内存、托管内存、进程 CPU 时间。
- 系统信息：机器名、操作系统、架构、.NET Runtime、环境名、处理器数量。
- CPU：采样当前进程 CPU，并按 Linux 或 macOS 口径读取系统 CPU。
- 内存：读取总内存、可用内存、系统已用内存和进程内存占比。
- 磁盘：通过 `DriveInfo.GetDrives()` 获取磁盘名称、类型、格式、总容量和可用空间。

系统监控接口需要 `Platform.SystemMonitor.View`。它适合定位“服务是否健康、资源是否接近瓶颈”，不替代 Prometheus、Grafana 这类长期监控系统。
