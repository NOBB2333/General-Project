# 审计链路

审计分为请求审计、登录活动、页面访问和实体审计。

## 请求审计

请求审计记录一次 HTTP 请求的入口信息，包括用户、租户、路径、方法、状态码、耗时和异常信息。

Host 运维租户时，请求审计还应记录运维会话和代入用户信息，用于回答“谁在什么租户上下文发起了这个请求”。

## 登录活动

登录活动记录账号最近登录时间和在线状态。它主要用于平台用户管理和在线用户视图，不应替代完整请求审计。

## 页面访问

页面访问记录前端路由访问。它适合分析页面使用情况，但不能作为安全审计的唯一依据。

## 实体审计

实体审计来自 ABP 的创建、修改、删除字段。它适合追踪业务数据生命周期，但在运维代入场景下，要和请求审计中的运维字段一起看。

## 排查建议

- 登录失败优先查认证接口和用户状态。
- 页面报 403 优先查角色权限和菜单权限。
- 页面报 500 优先查后端日志和请求审计异常。
- 运维租户操作优先查 `OperationSessionId`。
