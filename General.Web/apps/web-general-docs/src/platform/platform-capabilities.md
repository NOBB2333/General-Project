# 平台功能

平台中心不只是账号和菜单管理，它还提供系统运行时治理能力。这里按页面说明当前功能边界。

## 用户管理

用户管理负责租户内账号维护，包括用户列表、角色、组织归属、状态、密码和最近登录信息。

Host 管理员进入用户管理时，因为自身 `TenantId = null`，后端会走 Host 管理员分支，禁用多租户过滤后查询跨租户用户，并在 DTO 中返回租户名称。这是平台治理视角，不是把 Host 用户变成租户用户。

租户用户进入用户管理时，列表会受账号范围和组织数据范围影响。没有 `Platform.User.Manage` 时，用户可以只读查看页面，但新增、编辑、删除、重置密码、强制下线等管理动作不应出现。

## 组织架构

组织架构用于维护公司、部门和成员归属。默认展示范围来自当前用户的数据范围：

- 管理员可看全部组织。
- 组织负责人或 PM 可看本组织及下级。
- 普通成员默认只看自己相关组织。
- 组织新增、编辑、移动、成员转移和删除需要 `Platform.Organization.Manage`。

组织范围是用户管理、项目可见性和经营统计的重要基础。不要把组织树当成纯展示数据，它是权限计算的一部分。

## 角色权限

角色权限维护三类内容：

- 菜单权限：控制用户能进入哪些页面。
- 按钮权限：控制页面内的新增、编辑、删除、执行等动作。
- 数据授权：控制账号范围、组织范围和接口黑名单。

页面权限和管理权限应拆开。一个用户能进入租户列表或文件列表，不代表他能修改租户或删除文件。

## 菜单管理

菜单管理维护前端路由、菜单树、图标、排序和权限码。菜单记录既服务前端导航，也服务角色授权。

按钮也作为菜单树下的节点维护。这样角色授权时能把页面入口和页面内动作放到同一套模型里。

## 配置参数

配置参数用于在线维护少量可控系统设置。当前托管配置包括：

- 系统名称和登录页标题。
- 默认分页数。
- 定时任务记录保留数量。
- 审计日志保留天数。

配置值通过 ABP Setting 系统保存，支持全局配置和租户配置。不是所有环境变量都应该放到这里，数据库连接、密钥、外部服务凭证仍应走部署配置或安全存储。

## 缓存管理

缓存管理通过“缓存域版本号”刷新缓存，而不是直接扫描和删除所有缓存 key。

当前缓存域包括：

- `authorization`：角色、租户和接口访问授权缓存。
- `dict`：开放字典项缓存。
- `menu`：菜单、路由和权限树缓存。
- `notification`：通知相关缓存。
- `organization`：组织架构与成员范围缓存。
- `role`：角色与用户关系缓存。
- `tenant`：租户授权与租户状态缓存。

刷新某个缓存域时，系统更新该域版本号。后续业务读取会生成新的版本化 key，旧缓存自然失效。

## 字典管理

字典管理用于维护业务下拉、标签和状态项，例如项目状态、任务优先级、文件分类。

字典读取接口可以被业务页面复用，管理字典类型和字典项需要 `Platform.Dict.Manage`。业务模块不应硬编码这些枚举，除非它是代码级流程状态。

## 开放接口

开放接口用于给外部系统接入平台能力。页面维护外部应用、密钥、启停状态和授权信息。

它不是给普通前端页面用的菜单，而是用于系统间集成。典型场景包括外部 OA、财务系统、项目交付系统调用平台接口。

开放接口管理需要 `Platform.OpenApi.Manage`。密钥重置后应只展示一次，生产环境还需要配合 HTTPS、签名、IP 白名单和调用审计。

## 文件管理与回收中心

文件管理负责平台文件上传、下载、预览、元数据维护和批量删除。文件源配置决定文件存储方式。

回收中心用于恢复或彻底删除软删除数据。它属于高风险管理能力，应只授予管理员或专门运维角色。
