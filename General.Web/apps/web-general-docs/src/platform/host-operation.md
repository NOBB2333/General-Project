# Host 运维租户

Host 运维租户用于解决一个问题：平台管理员需要进入某个租户排查问题，但不能把操作责任记到一个没有实际操作的租户管理员身上。

## 设计目标

- Host 用户保持平台级身份。
- 运维请求带上目标租户上下文。
- 审计日志记录 Host 操作人、被代入用户、目标租户和运维会话。
- 不把普通租户用户和 Host 用户简单混在一起。

## 当前实现方向

进入运维租户时，系统签发带运维 claim 的 token。请求进入后，后端中间件读取这些 claim，并写入请求审计。

关键字段包括：

- `OperationSessionId`：本次运维会话。
- `OperationTenantId`：目标租户。
- `ImpersonatedUserId`：被代入的租户用户。
- `ImpersonatedUserName`：被代入的租户用户名。
- Host 操作人信息：记录真实发起请求的 Host 用户。

## 不做的事

- 不要求每个请求都手工传运维原因。
- 不把 Host 管理员永久加入租户用户表。
- 不把所有租户数据默认暴露给普通 Host 页面。

## 审计判断

排查一条运维操作时，优先看请求审计中的运维字段。实体创建人、修改人仍可能体现租户上下文，真实责任应结合运维会话判断。
