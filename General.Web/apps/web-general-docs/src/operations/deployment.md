# 构建与发布

项目提供完整构建脚本，用于生成后端、前端、迁移器和部署说明。

## 全量构建

```bash
./Script/package-all.sh
```

构建后会生成后端应用、数据库迁移器、前端静态资源和 Linux 脚本。

## 发布到 Linux

```bash
python3 ./Script/deploy-to-linux.py lan --init
```

`--init` 会同步后端和前端，执行迁移并重启服务。它不会自动删除远端 SQLite 数据文件。

## 数据迁移

服务器上可单独执行：

```bash
bash ./migrate-db.sh
```

迁移会执行 schema 更新和种子数据。已有数据会保留，种子逻辑负责补缺和修复默认演示数据。

## 注意事项

- `data/` 目录属于持久化数据，部署脚本默认不会覆盖。
- 如果要清空 SQLite，需要明确删除远端数据库文件。
- 生产环境应使用独立数据库和安全的连接字符串。
