# 排障手册

本页记录常见问题的定位顺序。

## 登录提示租户不存在

优先检查浏览器是否带了旧 token 或旧租户上下文。清理前端状态后仍异常，再查登录接口是否按用户名解析到了正确租户。

## 页面提示内部服务器错误

先看浏览器 Network 中具体失败接口。状态码含义：

- `401`：未登录或 token 失效。
- `403`：缺少接口权限。
- `404`：路由不存在或资源不可见。
- `500`：后端异常，应查日志。

## 项目列表为空

按顺序检查：

1. `AppProjects` 是否有项目。
2. `AppProjectMembers` 是否有成员关系。
3. 当前用户是否是 PMO、管理员、项目经理、发起人或项目成员。
4. 当前角色的数据范围是否覆盖项目组织。

## 经营总览为空

经营总览基于当前可见项目聚合。如果项目列表为空，经营总览也会为空。先修项目可见性，再查经营数据表。

## 租户页面只读报错

确认页面是否把辅助接口也绑定到了管理权限。只读页面需要查看权限，新增、编辑、删除才需要管理权限。
