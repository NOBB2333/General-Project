# 02 业务模块与管理规则

## 1. 业务模型

组织模型：

```text
集团
  └── 公司
       └── 部门
            └── 人员
```

项目模型：

```text
组合 Portfolio (多个项目集组合在一起 可选)
  └── 项目集 Program
       └── 项目 Project
```

项目执行模式：

- 瀑布模式：按阶段、里程碑、阶段评审、交付物推进
- 敏捷模式：按需求池、迭代、缺陷、版本推进
- 混合模式：项目主计划按阶段管理，研发执行按迭代管理

项目承接规则：

- 一个项目只有一个主责公司
- 一个项目可以有多个参与公司
- 默认由 PMO 指定主责公司
- 特殊情况可申请认领，但最终以 PMO 审核为准

## 2. 业务模块

系统业务能力按“共享平台能力 + 业务 APP 能力”来组织。

共享平台能力：

- 平台基础模块

业务 APP 能力：

- 项目管理模块
- 任务执行模块
- 研发交付模块
- PMO 管控模块
- 预算管理模块
- 合同执行模块
- 资料档案模块

对应关系建议如下：

- 项目执行 APP：项目管理模块、任务执行模块、研发交付模块、PMO 管控模块中的执行管控部分
- 经营管理 APP：预算管理模块、合同执行模块，以及经营类分析能力
- 智能分析 APP：后续承接 AI 摘要、风险提示、周报辅助生成、趋势洞察
- 平台中心：承接平台基础模块、资料档案模块和共享治理能力

- 项目阶段
- 里程碑
- 需求
- 迭代
- 任务
- 缺陷
- 测试计划
- 测试报告
- 发布记录
- 风险
- 问题
- 变更
- 会议纪要
- 行动项
- 决策记录
- 依赖项
- 假设项
- 预算
- 合同
- 资料台账
- 验收记录
- 开票记录
- 回款记录

任务对象建议字段：

- 编号
- 标题
- 描述
- 状态
- 优先级
- 所属项目
- 所属阶段
- 负责人
- 协作人
- 计划开始时间
- 计划结束时间
- 实际开始时间
- 实际结束时间
- 预估工时
- 实际工时
- 关联风险
- 关联里程碑

研发交付补充对象建议字段：

- 需求：编号、标题、来源、优先级、所属项目、所属迭代、状态、验收标准
- 迭代：名称、开始日期、结束日期、目标、状态、负责人
- 测试计划：编号、版本、测试范围、负责人、计划开始、计划结束、状态
- 发布记录：版本号、环境、发布时间、发布人、结果、回滚标记

治理补充对象建议字段：

- 会议纪要：会议类型、会议时间、参会人、纪要内容、所属项目、所属阶段或迭代
- 行动项：标题、责任人、来源纪要、计划完成时间、状态
- 决策记录：决策主题、背景、结论、决策人、生效时间、影响范围
- 依赖项：依赖类型、依赖对象、责任方、目标日期、状态

## 6. 管理规则

### 6.1 审批

一期统一以下审批：

- 项目立项审批
- 预算审批
- 预算调整审批
- 关键变更审批
- 合同新增或关键节点变更审批

### 6.2 通知

一期通知触发场景：

- 项目立项提交
- 审批待办生成
- 审批通过或驳回
- 任务指派
- 任务逾期
- 风险升级
- 问题超期
- 迭代开始或结束
- 测试计划临近
- 发布结果通知
- 里程碑临近或逾期
- 验收、开票、回款节点临近或逾期

### 6.3 权限

权限采用三层：

- 系统角色
- 组织角色
- 项目角色

数据范围原则：

- 上级可见本级和下级
- 下级不可反向查看上级
- 同项目成员默认可见项目日常执行数据
- 非项目成员默认不可见项目明细
- 研发执行数据默认在项目内共享，经营与资料敏感信息按字段和资料分类另行控制

敏感字段原则：

- 预算金额
- 合同金额
- 回款金额
- 利润
- 毛利率

这些字段默认仅 PM、PMO、公司管理员、集团管理员、财务接口人和被授权审批人可见。

资料访问原则：

- 合同、验收、开票、回款附件应支持单独授权
- 已归档资料默认只读
- 审计取证资料应限制下载和外发

### 6.4 审计

以下操作必须留痕：

- 状态变更
- 审批结果
- 金额变更
- 负责人变更
- 附件操作
- 导出操作
- 资料归档操作
- 发布结果确认
- AI 输出确认与采用

## 7. 看板与指标

### 7.1 APP 与看板对应关系

- 项目执行 APP：PM 工作台、PMO 执行总览、项目详情概览
- 经营管理 APP：预算预警、合同执行预警、验收 / 开票 / 回款预警
- 智能分析 APP：后续承接 AI 摘要、智能提示和趋势分析

### 7.2 PM 看板

展示重点：

- 我负责的项目
- 当前阶段
- 当前里程碑
- 当前迭代或当前版本
- 未完成任务
- 风险与问题
- 阻点
- 测试与发布状态
- 预算执行状态
- 合同执行节点
- 我的审批待办

### 6.2 PMO 总览看板

展示重点：

- 项目总数
- 进行中项目数
- 各阶段项目分布
- 延期项目数
- 风险项目数
- 预算偏差项目数
- 合同执行异常项目数
- 验收、开票、回款预警项目数

### 6.3 健康度

一期采用 6 维红黄绿模型：

- 进度健康
- 任务健康
- 风险健康
- 问题健康
- 预算健康
- 合同/回款健康

总规则：

- 任一维度为红，总体为红
- 无红但有黄，总体为黄
- 全绿才为绿

## 7. 一期规则边界

一期不展开：

- 复杂阶段门
- 复杂策略引擎
- 复杂预算模型
- 高级统计和预测
- 完整 DevOps 流水线编排
- 自由配置报表设计器
