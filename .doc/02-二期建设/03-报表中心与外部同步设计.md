# 项目执行视图与外部同步设计

## 1. 修订说明

本文件沿用原编号，当前改为二期项目执行区的数据视图与外部同步边界说明。

## 2. 二期重点

项目执行区二期的重点不是先做报表中心，而是：

- 把项目列表、项目详情、与我相关、PM 工作台、PMO 执行总览串起来
- 把项目执行对象和后续外部同步边界先理清

## 3. 外部同步建议

优先考虑后续可能同步的对象：

- 组织与用户主数据
- 研发工具链任务或缺陷摘要
- 工时或考勤相关数据

## 4. 同步原则

- 外部系统是主数据源时，本系统以同步和展示为主
- 不让外部同步牵着项目详情结构走
- 先保证内部对象稳定，再扩大同步范围

## 5. 当前结论

二期阶段最重要的是把项目执行区的数据主线先稳住，报表和外部同步只做边界预留。
