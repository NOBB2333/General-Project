# 报表中心与外部同步设计

## 1. 文档目的

本文件用于细化二期的报表中心基础版和外部系统单向同步设计。

## 2. 报表中心设计原则

- 报表中心优先做固定报表，不做自由拖拽报表平台
- 报表必须复用已有指标口径
- 报表输出应支持查看、筛选、导出三种基础能力

## 3. 报表中心基础版

### 3.1 建议首批报表

- 项目执行概览报表
- 项目风险与问题报表
- 预算执行报表
- 合同执行报表
- 验收 / 开票 / 回款报表
- 异常项目报表

### 3.2 页面要求

- 支持固定筛选条件
- 支持列表展示
- 支持汇总区与明细区
- 支持导出

## 4. 报表筛选维度

建议支持以下筛选：

- 时间范围
- 公司 / 部门
- 项目分类
- 项目经理
- 项目状态
- 重点项目标记

## 5. 外部同步对象

建议二期重点同步以下对象：

- 组织和用户主数据
- 预算执行金额
- 合同状态
- 开票状态
- 回款状态

## 6. 同步原则

- 外部系统是主数据源时，本系统只读同步
- 同步失败不阻断主业务，但必须可追踪
- 每类同步对象都要定义频率和失败补偿策略

## 7. 同步频率建议

- 组织与用户：按日同步或事件触发同步
- 财务执行金额：按日或按结算批次同步
- 开票与回款状态：按日同步

## 8. 失败处理原则

- 记录同步任务日志
- 记录失败对象和失败原因
- 支持手工重试
- 支持失败告警

## 9. 本文件结论

二期的报表中心和外部同步不追求平台化，而是先把固定报表和稳定同步机制做扎实。
