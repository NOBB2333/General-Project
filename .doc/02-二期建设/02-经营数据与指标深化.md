# 项目执行数据与指标深化

## 1. 修订说明

本文件沿用原编号，当前改为二期项目执行区的数据与指标设计。

## 2. 核心数据对象

- 项目
- 里程碑
- 周期
- 任务
- 问题
- RAID 项
- 文档
- 成员
- 工时

## 3. 核心指标

### 3.1 项目详情

- 项目状态
- 优先级
- 成员规模
- 任务完成率
- 高风险事项数
- 未闭环问题数

### 3.2 PM 工作台

- 我的项目数
- 我的任务数
- 我的问题数
- 本周重点事项

### 3.3 PMO 执行总览

- 进行中项目数
- 重点项目数
- 异常项目数
- 逾期里程碑项目数

## 4. 口径原则

- 所有指标必须能进入项目详情明细
- 指标由后端统一定义
- 前端负责展示，不各自再拼口径

## 5. 当前结论

二期数据深化的重点不是对象越多越好，而是把项目执行区最关键的对象和指标先做稳定。
