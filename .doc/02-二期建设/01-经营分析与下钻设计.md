# 项目执行区信息架构与下钻设计

## 1. 修订说明

本文件沿用原文件编号，但当前内容按 `二期 = 项目执行区` 重新定义。

## 2. 二期目标

二期的目标不是先做经营分析，而是把项目执行区真正做成系统主业务区。

核心入口应收敛为：

- 项目列表
- 项目详情
- 与我相关
- PM 工作台
- PMO 执行总览

## 3. 下钻主链路

### 3.1 PMO 视角

```text
PMO 执行总览
  -> 异常项目列表
  -> 单项目详情
  -> 指定 Tab 明细
```

### 3.2 PM 视角

```text
PM 工作台
  -> 我的项目
  -> 项目详情
  -> 任务 / 问题 / 文档 / 人员明细
```

### 3.3 成员视角

```text
与我相关
  -> 我的项目
  -> 我的任务 / 我的问题 / 我的工时
  -> 对应项目详情
```

## 4. 当前设计原则

- 项目详情必须是项目执行区核心页
- 所有汇总结果都必须能进入项目详情
- 下钻进入后要能定位具体 Tab

## 5. 当前结论

二期真正的“分析与下钻”，核心不是经营图表，而是项目执行驾驶舱里的汇总到明细链路。
