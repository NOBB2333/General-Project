# 经营分析与下钻设计

## 1. 文档目的

本文件用于细化二期的经营分析能力和从汇总进入明细的下钻链路。

本文件不重复说明二期“做什么范围”，分期范围统一以顶层 `04` 为准。

## 2. 设计原则

- 二期分析建立在一期执行数据稳定基础上
- 所有汇总分析都必须能进入明细
- 所有指标必须能追溯到项目、合同、节点或业务记录
- 分析能力优先服务 PMO 和管理层，其次服务 PM

## 3. 预算分析增强

### 3.1 分析目标

- 让预算不只是可录入、可预警，还能看执行趋势和偏差来源

### 3.2 建议分析内容

- 预算总额、已执行金额、剩余预算
- 预算执行率趋势
- 预算偏差率趋势
- 按部门、项目经理、项目分类的预算对比
- 预算调整前后对比

### 3.3 下钻链路

```text
预算总览
  -> 预算异常项目列表
  -> 单项目预算详情
  -> 预算科目或调整记录
```

## 4. 合同执行增强

### 4.1 分析目标

- 让合同不只是项目挂接信息，而是可以反映履约进度和经营节点状态

### 4.2 建议分析内容

- 合同总数、执行中合同数、已完成合同数
- 合同金额分布
- 合同节点完成率
- 合同逾期节点分布
- 多合同项目的合同执行对比

### 4.3 下钻链路

```text
合同执行总览
  -> 合同列表
  -> 合同详情
  -> 履约 / 验收 / 开票 / 回款节点明细
```

## 5. 验收、开票、回款分析

### 5.1 分析目标

- 让项目经营节点形成完整分析链条，而不只是零散记录

### 5.2 建议分析内容

- 验收完成率
- 开票完成率
- 回款完成率
- 节点逾期率
- 月度验收、开票、回款趋势
- 按项目、部门、项目经理的节点完成情况

### 5.3 下钻链路

```text
经营节点总览
  -> 项目经营列表
  -> 单项目经营详情
  -> 验收 / 开票 / 回款记录明细
```

## 6. 健康度综合分析

### 6.1 分析目标

- 把执行风险和经营风险统一到项目健康度中

### 6.2 建议纳入的分析信号

- 里程碑逾期
- 高风险未关闭
- 超期问题
- 预算偏差
- 合同节点逾期
- 验收 / 开票 / 回款节点逾期

### 6.3 展示建议

- 健康度分级分布
- 红黄项目列表
- 健康度趋势变化
- 从健康度结果进入异常原因明细

## 7. 图表下钻设计

### 7.1 下钻基本路径

- 指标卡 -> 异常项目列表
- 趋势图 -> 指定时间范围列表
- 分类对比图 -> 指定分类项目列表
- 项目列表 -> 项目详情页

### 7.2 下钻统一要求

- 下钻后的筛选条件自动继承
- 汇总页与明细页口径一致
- 导出结果与当前筛选结果一致

## 8. 页面形态建议

建议二期新增或增强以下页面：

- 预算分析页
- 合同执行分析页
- 经营节点分析页
- 健康度分析页
- 报表明细列表页

## 9. 本文件结论

二期分析能力的重点不是图更多，而是把预算、合同、经营节点和健康度真正串成一条从汇总到明细的分析链路。
