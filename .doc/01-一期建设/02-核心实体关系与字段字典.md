# 核心实体关系与字段字典

## 1. 修订说明

一期按平台中心重写后，核心实体也先聚焦平台底座对象。

## 2. 一期核心实体

- 组织
- 用户
- 角色
- 菜单
- 角色菜单关系
- 文件
- 审计日志
- 定时任务

## 3. 核心关系

```text
组织
  └── 用户
        └── 角色
              └── 菜单

文件 -> 关联业务对象
审计日志 -> 记录操作对象与操作者
定时任务 -> 记录调度配置与执行状态
```

## 4. 最小字段建议

### 4.1 组织

- 组织编码
- 组织名称
- 上级组织
- 排序
- 状态

### 4.2 用户

- 用户名
- 姓名
- 所属组织
- 角色列表
- 状态
- 最后登录时间

### 4.3 角色

- 角色编码
- 角色名称
- 状态
- 说明

### 4.4 菜单

- 菜单编码
- 菜单名称
- 菜单路径
- 上级菜单
- 图标
- 排序
- 是否启用

### 4.5 文件

- 文件名
- 所属业务对象类型
- 所属业务对象 ID
- 上传人
- 上传时间
- 存储路径

### 4.6 审计日志

- 操作人
- 操作时间
- 操作对象
- 操作类型
- 请求结果

## 5. 当前结论

一期字段设计的重点不是一次把所有业务对象建完，而是先把平台治理对象设计稳定。
