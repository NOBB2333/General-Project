# 核心实体关系与字段字典

## 1. 文档目的

本文件用于补齐一期原型、接口、数据库建模所需的核心实体草案和最小字段字典。

## 2. 核心实体总览

一期建议把实体分成七类：

- 基础类：组织、部门、用户、角色
- 项目类：项目集、项目、项目模板、阶段、里程碑
- 执行类：需求、迭代、任务、缺陷、待办、工时
- 交付类：测试计划、测试报告、发布记录
- 管控类：风险、问题、变更、审批、通知、评论、附件、日志
- 治理类：会议纪要、行动项、决策记录、依赖项、假设项
- 资料类：资料台账、资料版本、归档记录
- 经营类：预算、预算调整、合同、验收记录、开票记录、回款记录

## 5. 外部系统主数据来源建议

建议一期按以下原则处理主数据来源：

- 用户、部门、组织：优先来自 SSO / AD / LDAP
- 财务执行金额：可来自 ERP / 财务系统，也可先手工补录
- 消息通道：来自企业微信 / 钉钉 / 飞书 / 邮件

## 6. 字段设计约束

一期字段设计建议统一遵守以下规则：

- 每个核心对象都要有编码
- 每个核心对象都要有状态
- 每个核心对象都要有创建人、创建时间、更新时间
- 审批相关对象都要保留审批意见
- 敏感金额字段必须支持字段级权限控制
- 用于统计的字段应有明确口径，避免前后端各算一套
- 资料对象应支持版本和归档状态
- 研发交付对象应能挂接项目、阶段或迭代
