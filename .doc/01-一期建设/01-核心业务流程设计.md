# 核心业务流程设计

## 1. 修订说明

一期当前重新定义为 `平台中心建设阶段`，因此本文件不再展开项目执行和经营管理的详细流程，而是聚焦系统底座流程。

## 2. 一期核心流程总览

一期必须先跑通以下平台级流程：

1. 用户登录与认证流程
2. 组织与用户维护流程
3. 角色授权流程
4. 菜单配置与菜单授权流程
5. 文件上传与留痕流程
6. 审计日志记录与查询流程
7. 系统监控与任务巡检流程

## 3. 用户登录与认证流程

```text
用户输入账号密码
  -> 登录校验
  -> 获取身份与权限
  -> 获取菜单
  -> 进入系统首页
```

规则：

- 登录后先确定用户身份和角色
- 再决定可见菜单和可访问页面
- 菜单权限与页面权限必须保持一致

## 4. 组织与用户维护流程

```text
创建组织
  -> 创建用户
  -> 分配所属组织
  -> 分配角色
  -> 生效
```

规则：

- 用户不能脱离组织体系长期孤立存在
- 角色分配应以岗位和职责为依据

## 5. 角色与菜单授权流程

```text
创建角色
  -> 选择菜单
  -> 配置权限
  -> 绑定用户
  -> 用户刷新后生效
```

规则：

- 权限设计优先用角色承接
- 菜单不是摆设，必须和权限一致

## 6. 文件与审计留痕流程

```text
上传文件
  -> 记录上传人和时间
  -> 关联业务对象
  -> 后续查看 / 下载 / 导出继续留痕
```

```text
发生系统操作
  -> 记录操作日志
  -> 进入审计查询
  -> 支持后续追踪
```

## 7. 当前结论

一期真正的核心流程不是业务对象越多越好，而是先把平台底座流程做稳，让后续项目执行和经营管理都能落在统一规则上。
